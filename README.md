# [SIL] vECU Development

## Overview
This project focuses on the development and validation of a virtual Electronic Control Unit (vECU) in a Software-in-the-Loop (SIL) environment. It enables early-stage testing, debugging, and verification of automotive software before deployment to target hardware.

## Key Features
- Virtual ECU implementation
- Software-in-the-Loop (SIL) testing
- CAN communication simulation
- Software validation and verification
- Automotive embedded software development

# Technical Skills

* **Programming & Scripting:** C, Embedded C, Python
* **Automotive Tools & SIL/HIL:** Synopsys Silver (vECU & SIL simulation), TIS, ECU Test, TestFlow, INCA, MDA, ECU Worx, AutoSPY (calibration & trace analysis), SDOM, CI Dashboard, RequestOne, CANoe.
* **Embedded Systems & RTOS:** ARM Cortex-M Microcontrollers (Cortex-M4, ARM7TDMI), UART, SPI, I²C, GPIO
* **Communication Protocols:** CAN, LIN, Ethernet (automotive), SPI, UART, I²C
* **AUTOSAR & Configuration:** AUTOSAR BSW understanding, ARXML review, BSW module configuration, High cut / Low-cut configurations for Complex Drivers.
* **SIL / HIL Validation:** vECU development & SIL setup, HIL test execution and debugging, Virtual Network Plug (VNP), virtual model integration
* **Debugging Tools:** Keil, Oscilloscope, Logic Analyzer, INCA / MDA / AutoSPY trace debugging
* **Version Control & Workflow:** Git, GitHub, SVN, Jira (test management, issue tracking)
* **Model-Based Development:** MATLAB, Simulink, CAPL scripting (Vector tools)
* **Standards & Methodologies:** ASPICE (working knowledge), MISRA C (familiarity), SDLC, V-Model workflows
